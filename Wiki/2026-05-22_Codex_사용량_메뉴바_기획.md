## 추천 접근

처음부터 큰 앱으로 가지 말고 두 단계로 간다.

1. 데이터 extractor를 먼저 만든다.
   - 입력: `~/.codex/sessions`
   - 출력: `{ primaryUsed, primaryResetAt, secondaryUsed, secondaryResetAt, stale }`
   - 기준: JSONL에서 `type=event_msg`이고 `payload.rate_limits`가 있는 마지막 line만 본다.

2. 빠른 메뉴바 prototype을 붙인다.
   - 가장 빠른 검증은 SwiftBar script다.
   - 실제 제품은 SwiftUI `MenuBarExtra` 기반 native app으로 만든다.

최종 추천은 native macOS app이다. 메뉴바 label은 AppKit `NSStatusItem` custom view로 직접 그리고, 클릭 상세는 SwiftUI popover로 관리한다.

## 위험과 대응

- Local log schema가 바뀔 수 있다.
  - 대응: parser를 독립 모듈로 두고, `rate_limits`가 없으면 stale/error UI와 공식 Usage 링크를 보여준다.

- 로그 파일에 prompt 내용도 들어 있다.
  - 대응: 앱은 JSON line 전체를 저장하지 않고 `rate_limits` 객체만 메모리에 남긴다.

- 세션이 새로 시작될 때 최신 파일 탐색이 필요하다.
  - 대응: `~/.codex/sessions` 하위의 mtime 최신 JSONL을 우선 보고, 필요 시 오늘/어제 폴더까지 스캔한다.

## 2026-05-22 Compact Battery UI 수정

- 위 줄: `5h` 남은 비율과 battery-style bar.
- 아래 줄: `1w` 남은 비율과 battery-style bar.
- Native app refresh interval: 30초에서 60초로 변경.
- Bar fill 기준: 사용률이 아니라 남은 비율.
- 색상 기준: 남은 비율 10% 이하 노랑, 5% 이하 빨강, 그 외 초록.
- Popover도 header를 줄이고 두 usage row 중심으로 재정리했다.

## 2026-05-22 MenuBarExtra 표시 문제 수정

실제 메뉴바에서 SwiftUI `MenuBarExtra` label의 `VStack + GeometryReader`가 제대로 렌더링되지 않고 첫 줄 `5h`만 크게 잘려 보이는 문제가 확인됐다. 메뉴바 label은 AppKit `NSStatusItem` custom `NSView`로 교체했고, 두 줄 텍스트와 얇은 bar를 `draw(_:)`에서 직접 그리도록 변경했다.

- 메뉴바 표시: AppKit `NSStatusItem` custom view.
- 클릭 상세: 기존 SwiftUI popover 유지.
- 빌드/테스트: `swift test`, `Scripts/build-app.sh`, `plutil -lint` 통과.
- 앱 재실행: 기존 process 종료 후 `dist/Howmuchusage.app` 재실행 확인.

## 2026-05-22 남은양 숫자와 Open Usage 링크 수정

사용자 피드백으로 메뉴바에 보이는 숫자가 `used_percent`라서 “남은양” 요구와 반대로 보이는 문제가 확인됐다. 수정 후 visible percent는 모두 `100 - used_percent` 기준이다.

- 메뉴바 두 줄 숫자: 사용률이 아니라 남은 비율 표시.
- SwiftBar/text 출력: `used` 문구 제거, `left` 기준으로 표시.
- Popover row: 큰 숫자는 남은 비율만 표시하고 reset 정보를 보조 정보로 배치.
- 정렬: AppKit status item 폭을 넓히고 `label / bar / percent` column 위치를 고정해 두 줄 grid가 맞도록 조정.
- Open Usage: `https://chatgpt.com/codex/usage`에서 `https://chatgpt.com/codex/settings/usage`로 변경.
