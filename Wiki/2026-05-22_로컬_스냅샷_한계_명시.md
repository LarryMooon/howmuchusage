# 2026-05-22 로컬 스냅샷 한계 명시 작업

## 배경

사용자 판단:

- Howmuchusage가 읽는 `rate_limits` 값은 Codex가 로컬 session log에 기록한 스냅샷이다.
- 사용자가 Codex 공식 Usage panel을 열거나 Codex가 usage 정보를 새로 받는 순간이 아니면 local log가 갱신되지 않을 수 있다.
- 따라서 Howmuchusage의 reload는 새 서버 사용량 조회가 아니라 이전 local snapshot을 다시 읽는 동작일 수 있다.

이 판단이 현재 구조와 맞다. 앱의 역할을 “실시간 usage fetcher”가 아니라 “local snapshot viewer”로 명확히 잡는다.

## 변경

- 메뉴바 label을 `5h`/`1w`에서 `~5h`/`~1w`로 변경했다.
  - `~`는 official live value가 아니라 local approximate snapshot이라는 의미다.
- local snapshot이 10분 이상 오래되면 메뉴바 text와 bar를 회색으로 표시한다.
- popover 상단에 `Local snapshot · Xm ago`를 강조 표시한다.
- `Refresh` 버튼 이름을 `Reload Snapshot`으로 변경했다.
- popover 하단에 `Reload only re-reads local logs. Official Usage may differ.` 안내를 추가했다.
- README의 Accuracy section을 Limitations 중심으로 보강했다.
- version 기본값을 `0.1.2`, build number를 `3`으로 올렸다.

## 한계 문구 기준

앱은 다음을 명확히 말해야 한다.

- 이 앱은 official OpenAI usage API client가 아니다.
- `Reload Snapshot`은 `~/.codex/sessions` local log를 다시 읽을 뿐이다.
- Reload는 Codex 또는 OpenAI server가 새 `rate_limits`를 쓰도록 강제하지 못한다.
- Codex 공식 Usage panel과 값이 다르면 공식 panel을 기준으로 본다.
