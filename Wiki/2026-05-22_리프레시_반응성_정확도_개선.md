# 2026-05-22 리프레시 반응성/정확도 개선 작업

## 문제

사용자 피드백:

- 메뉴바 앱이 빠릿빠릿하게 반응하지 않는다.
- `Refresh`를 눌러도 바로 반응하는 느낌이 약하다.
- Codex 공식 Usage panel과 표시값이 다르다.

## 확인한 원인

1. `Refresh`가 main actor에서 바로 `CodexUsageReader.latestSnapshot()`을 실행했다.
   - session log 파일 스캔과 JSON decode가 UI thread를 점유했다.
   - 이미 빌드된 debug probe 기준 수정 전 read 시간이 약 1.77초였다.
2. reader가 JSONL tail 안의 모든 줄을 decode했다.
   - 실제 필요한 값은 `payload.rate_limits`뿐인데, 큰 response/encrypted payload line까지 decode 후보가 됐다.
3. 앱의 데이터 원천은 공식 OpenAI Usage API가 아니라 local `~/.codex/sessions` log다.
   - `Refresh`는 서버에서 최신 사용량을 가져오는 버튼이 아니라 local log를 다시 읽는 버튼이다.
   - 따라서 공식 Codex Usage panel과 값이 다르면 공식 panel을 우선해야 한다.

## 변경

- `UsageViewModel.refresh()`를 background task 기반으로 변경했다.
- refresh 중에는 popover 버튼이 `Refreshing...`으로 바뀌고 비활성화된다.
- popover 상세에 `Refresh` 상태와 `Mode: local Codex session log`를 추가했다.
- `CodexUsageReader`는 `"rate_limits"`가 없는 line은 JSON decode하지 않는다.
- 최신 snapshot을 찾은 뒤, 다음 file의 modified time이 이미 best snapshot보다 오래됐으면 더 오래된 파일 scan을 중단한다.
- README의 Accuracy 설명을 보강했다. `Refresh`는 OpenAI server-side usage endpoint를 호출하지 않는다고 명시했다.
- version 기본값을 `0.1.1`, build number를 `2`로 올렸다.
