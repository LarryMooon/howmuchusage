# 2026-05-22 배포 릴리즈 작업 기록

## 작업 목표

Howmuchusage를 다른 Mac 사용자에게 zip으로 전달할 수 있는 형태로 만들고, 공개 배포에 필요한 code signing/notarization 경로와 Launch at Login option을 추가한다.

## 공식 기준

- Apple은 macOS 10.15 이후 기본 Gatekeeper 통과를 위해 외부 배포 앱을 Apple-issued Developer ID certificate로 sign하고 notarize해야 한다고 안내한다.
- Apple notarization 문서는 zip archive 제출, `notarytool`, `stapler` 사용을 안내한다.
- Apple ServiceManagement 문서는 macOS 13+에서 login item 제어에 `SMAppService`를 사용하라고 안내한다.

## 구현 내용

### 실행 피드백

- 앱은 `LSUIElement` 메뉴바 앱이라 Finder에서 더블클릭해도 Dock icon이나 일반 window가 뜨지 않는다.
- 이 동작이 "아무 일도 안 일어남"처럼 보일 수 있어 첫 실행 직후 `showPopoverSoon()`으로 메뉴바 popover를 자동 표시한다.
- 이미 실행 중인 앱을 Finder에서 다시 열 때도 `applicationShouldHandleReopen`에서 popover를 다시 표시한다.

### Launch at Login

- `HowmuchusageMenuBarApp.swift`에 `ServiceManagement` import를 추가했다.
- `UsageViewModel`이 `SMAppService.mainApp.status`를 읽어 `On`, `Off`, `Needs approval`, `Move to Applications` 상태를 표시한다.
- Popover에 `Launch at Login` toggle을 추가했다.
- 등록/해제는 `SMAppService.mainApp.register()`와 `unregister()`를 사용한다.

### App bundle build

- `Scripts/build-app.sh`를 배포용으로 확장했다.
- 지원 환경변수:
  - `APP_NAME`
  - `BUNDLE_ID`
  - `VERSION`
  - `BUILD_NUMBER`
  - `DIST_DIR`
  - `UNIVERSAL`
- 기본 bundle id는 `com.larrymoon.howmuchusage`다.
- `UNIVERSAL=1`이면 SwiftPM `--arch arm64 --arch x86_64`로 universal binary를 만든다.

### Release package

- `Scripts/package-release.sh`를 추가했다.
- 기본 실행은 universal build 후 `dist/release/Howmuchusage-0.1.0-universal-macos.zip`과 `.sha256`을 만든다.
- GitHub Release asset 업로드가 Chrome file upload 권한에 막힐 수 있으므로, 공개 repo 배포용으로 같은 파일을 `Downloads/` 폴더에도 복사한다.
- `CODESIGN_IDENTITY`가 있으면 Developer ID signing을 수행한다.
- `NOTARIZE=1`이면 `notarytool submit --wait`, `stapler staple`, `stapler validate`를 이어서 실행한다.
- Notary credential은 `NOTARY_PROFILE` 또는 `NOTARY_APPLE_ID`/`NOTARY_TEAM_ID`/`NOTARY_PASSWORD` 환경변수를 지원한다.

## 남은 외부 의존성

로컬 환경에는 Developer ID Application certificate와 notarization credential이 없어서 실제 Apple notarization 제출은 수행하지 않았다. 공개 배포 시에는 아래 값이 필요하다.

```sh
CODESIGN_IDENTITY="Developer ID Application: Your Name (TEAMID)"
NOTARIZE=1
NOTARY_PROFILE=howmuchusage-notary
```

또는:

```sh
NOTARY_APPLE_ID="apple-id@example.com"
NOTARY_TEAM_ID="TEAMID"
NOTARY_PASSWORD="app-specific-password"
```

## 사용 예시

개발용 zip:

```sh
Scripts/package-release.sh
```

공개 배포용 zip:

```sh
xcrun notarytool store-credentials howmuchusage-notary \
  --apple-id "apple-id@example.com" \
  --team-id "TEAMID" \
  --password "app-specific-password"

CODESIGN_IDENTITY="Developer ID Application: Your Name (TEAMID)" \
NOTARIZE=1 \
NOTARY_PROFILE=howmuchusage-notary \
  Scripts/package-release.sh
```
