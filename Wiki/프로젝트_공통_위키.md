# Project Howmuchusage 공통 Wiki

## 프로젝트 목적

Mac 메뉴 바에서 Codex 남은 사용 가능량을 작고 빠르게 확인하는 개인용 도구를 만든다. 목표 표시는 현재 5시간 세션 남은 비율, 5시간 reset까지 남은 시간, 주간 남은 비율, 주간 reset 시각이다.

## 현재 핵심 결정

- 최종 제품 방향은 macOS native menubar app이다.
- 구현 전 데이터 검증은 간단한 CLI 또는 SwiftBar script로 먼저 한다.
- 사용량 데이터의 1차 원천은 `~/.codex/sessions/YYYY/MM/DD/*.jsonl` 안의 `event_msg.payload.rate_limits`다.
- `rate_limits.primary`는 5시간 window로 보고, `window_minutes=300`, `used_percent`, `resets_at`을 사용한다.
- `rate_limits.secondary`는 주간 window로 보고, `window_minutes=10080`, `used_percent`, `resets_at`을 사용한다.
- OpenAI 공식 문서의 Codex Usage panel은 사용자가 직접 확인하는 기준 화면과 fallback 검증용으로 둔다.

## 확인된 로컬 증거

- 현재 프로젝트는 새 프로젝트에 가깝고, 기존 source-of-truth Wiki가 없었다.
- 현재 로컬 세션 예시:
  - 파일: `/Users/larrymoon/.codex/sessions/2026/05/22/rollout-2026-05-22T19-18-31-019e4f31-9a6b-7c61-95dc-bb7c233ad984.jsonl`
  - line 88 기준 `primary.used_percent=49.0`, `primary.window_minutes=300`, `primary.resets_at=1779457603`
  - line 88 기준 `secondary.used_percent=53.0`, `secondary.window_minutes=10080`, `secondary.resets_at=1779848664`
  - KST 변환: 5시간 reset `2026-05-22 22:46:43 KST`, 주간 reset `2026-05-27 11:24:24 KST`

## 공식 문서 기준

- OpenAI Help Center의 [Using Codex with your ChatGPT plan](https://help.openai.com/en/articles/11369540-using-codex-with-your-chatgpt-plan.pdf)는 Codex 사용량 제한이 plan과 작업 복잡도에 따라 달라지고, 한도에 가까워지면 Codex usage page 또는 limit banner를 확인하라고 안내한다.
- OpenAI Help Center의 [Codex rate card](https://help.openai.com/en/articles/20001106-codex-rate-card)는 Codex settings > Usage panel에서 usage limits와 remaining credit을 볼 수 있다고 안내한다.
- 이 프로젝트의 local log reader는 편의용 보조 도구이며, OpenAI의 공식 Usage panel과 다를 경우 공식 화면을 우선한다.

## 제품 원칙

- 메뉴 바 본문은 짧아야 한다. 현재 기준은 두 줄 표시다: 위 `5h`, 아래 `1w`.
- 각 줄은 남은 percent와 battery-style 얇은 가로막대를 함께 보여준다. 숫자와 막대 fill은 모두 남은 비율 기준이다.
- 색상 기준은 남은 비율 기준으로 10% 이하 노랑, 5% 이하 빨강, 그 외 초록이다.
- 클릭했을 때만 상세를 보여준다: 5시간 남은 비율, reset 시각, 남은 시간, 주간 남은 비율, 주간 reset 시각, 마지막 업데이트, 데이터 원천.
- stale data는 확실히 구분한다. 최근 `rate_limits`가 10분 이상 갱신되지 않으면 메뉴 바에 흐린 상태 또는 `--`를 표시한다.
- 내부 로그 파일을 읽을 때 prompt 내용이나 대화 본문은 표시하거나 저장하지 않는다. `rate_limits` 객체만 파싱한다.
- schema 변경 가능성을 전제로 parser를 작게 유지하고, 실패 시 공식 Usage panel 링크를 보여준다.

## 구현 상태

- SwiftPM package로 구현했다.
- 핵심 parser는 `Sources/CodexUsageCore/`에 있다.
- CLI는 `howmuchusage-probe` product이며 JSON, text, SwiftBar format을 지원한다.
- SwiftBar script는 `Scripts/codex-usage.1m.sh`다.
- Native 메뉴바 앱은 `HowmuchusageMenuBar` product이며 AppKit `NSStatusItem` custom view로 메뉴바의 두 줄 battery 표시를 직접 그린다. 클릭 상세 popover는 SwiftUI를 사용한다. 1분마다 자동 refresh한다.
- Launch at Login toggle은 macOS 13+ `SMAppService.mainApp`로 구현한다.
- App bundle 생성은 `Scripts/build-app.sh`로 수행하며 결과물은 `dist/Howmuchusage.app`에 생성된다. 기본 bundle id는 `com.larrymoon.howmuchusage`다.
- 배포 zip 생성은 `Scripts/package-release.sh`로 수행한다. 기본은 universal build이며 `dist/release/Howmuchusage-0.1.0-macos.zip`과 SHA-256 checksum을 만든다.
- 공개 배포용 signing/notarization은 `CODESIGN_IDENTITY`, `NOTARIZE=1`, `NOTARY_PROFILE` 또는 Apple ID credential 환경변수를 사용한다.

## 검증 명령

```sh
swift build -c release --product howmuchusage-probe
Scripts/codex-usage.1m.sh
Scripts/build-app.sh
Scripts/package-release.sh
open dist/Howmuchusage.app
```

## 2026-05-22 남은양 표시 보정

- 사용자 피드백으로 메뉴바 숫자가 사용률처럼 보이는 문제가 확인됐다.
- `used_percent`는 parser 내부 계산값으로만 두고, 메뉴바/SwiftBar/text/popover의 visible percent는 `100 - used_percent`인 남은 비율로 통일했다.
- 메뉴바 status item 폭과 column 위치를 조정해 `5h`/bar/percent, `1w`/bar/percent가 같은 grid에 정렬되도록 했다.
- Open Usage URL은 404가 나는 `/codex/usage`에서 `https://chatgpt.com/codex/settings/usage`로 변경했다. 공식 문서 표현은 Codex settings > Usage panel이다.
